说话人身份识别
方案1：先分离再ASR：
1. 如果开启说话人识别
2. 则先通过说话人分离
3. 再进行语音识别
4. 再通过当前分离的说话人向量进行相似度查询，获取N个top且大于某阈值的声纹记录
5. 如有结果，找到最佳阈值记录并替换说话人在数据库中的真实身份
6. 没找到默认使用分离后的说话人
方案2：段落级打标签
你现在的 assign_speakers_to_segments() 是 “段落级别分配说话人”，流程是这样的：
- Whisper 识别后，返回多个 segment，每个 segment 代表一段连续语音。
- 然后用 diarization 分离的结果（时间+speaker 标签）按时间重叠（交集时长最大），给每个 segment 打一个 speaker。
✅ 优点：
- 简单快速
- 不需要 words 级别的时间戳
- 对大多数应用（字幕、会议摘要等）已足够
⚠️ 缺点：
- 如果一个段落中有多个人在说话（例如切换较快），可能会误分为同一个人
- 无法精细地分配 speaker 到单词级别，缺乏粒度
方案3：逐词级打标签
这是 WhisperX 的做法，也支持：
每个 segment 下包含多个 word（每个 word 有 start 和 end 时间），我们可以给每个 word 打上一个 speaker。
实现流程：
1. 遍历 segment["words"] 里每个 word
2. 用 diarization 的结果去匹配 word 的时间区间
3. 找出与该 word 交集时间最长的 speaker，赋值给 word["speaker"]
5分钟语音速度对比：
asr(1x) < 方案2≈方案3(3x) <方案1（4-5x）
准确性简单对比：
方案1比较依赖pyannote.audio的分离效果，分离太碎 asr的时候容易丢失句子语义
方案2比较依赖asr+vad模型的识别效果，如果一句中有多人说话只能打标时间最长者，句子语义保留的比较好
方案3比较类似方案2，也无法解决多人说话的问题，句子语义保留的比较好
