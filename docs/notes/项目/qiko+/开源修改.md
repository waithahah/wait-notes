# Qiko v1.1.0 用户权限说明

## 用户体系
### 租户
使用`注册`功能将注册一个租户，暂时只支持使用`邮箱`注册，每个租户可以拥有一个个人工作空间(后续简称`个人空间`) + 若干其他租户邀请使用的工作空间(后序简称`受邀空间`)。
> 权限控制在`权限控制`模块描述。
### 用户
一个租户可在`设置 -> 成员`邀请多个用户，暂时仅支持租户通过邀请链接注册用户，用户类型有：
- 管理员
- 普通用户
> 权限控制在用户权限模块描述。

## 权限控制
### 租户权限
注册入口可根据情况控制是否开放。
> 开放入口：`NEXT_PUBLIC_EDITION=SELF_HOSTED`通过设置前端环境变量控制。
* 个人空间
  
    租户拥有最高权限，可使用工作空间中所有内容，可管理模型和工具。   

* 受邀空间

    受邀空间根据受邀角色不同分为下述两类：
    1.  受邀为管理员：可使用工作空间中所有内容，可管理模型和工具
    2.  受邀为普通用户：仅可使用应用程序，不可使用其他管理性设置

# Qiko v1.1.0 用户权限实现说明
## 注册

### 相关表
| 表名                 | 说明       |
| -------------------- | ---------- |
| tenants              | 租户表     |
| accounts             | 用户表     |
| tenant_account_joins | 租户用户表 |
### 注册租户逻辑
在租户注册页面注册后，会创建一条`tenants`记录，一条`accounts`表记录，一条`tenant_account_joins`表记录。 
### 注册租户邀请的用户逻辑
在租户邀请用户页面邀请用户后，会创建一条`accounts`表记录，一条`tenant_account_joins`表记录。

### 密码加密规则
生成一个16字节的随机盐值，并将其进行base64编码得到base64_salt。然后，使用一个名为hash_password的函数，该函数用于对给定的密码字符串和盐值进行加密，使用的是PBKDF2HMAC算法，经过10000次迭代运算，返回加密后的密码的十六进制表示形式。将密码和盐值进行加密得到password_hashed，再将其进行base64编码得到base64_password_hashed。最后，将base64_password_hashed和base64_salt分别赋值给账号对象的password和password_salt属性，完成密码加密存储的过程。
