我明白了，您想要的是只拉取远程仓库的主分支，并且不希望本地保留其他远程分支的信息。这可以通过以下步骤实现：

拉取远程主分支：使用git fetch origin \<main-branch>\命令只拉取远程仓库的主分支（例如master或main）的更新。

bash
git fetch origin master
创建本地分支：基于远程主分支，创建一个本地分支。这将使您能够在本地进行更改并提交到远程主分支。

bash
git checkout -b master origin/master
删除其他远程分支引用：如果您不需要其他远程分支的引用，可以使用以下命令来清理本地不再需要的远程分支引用。请确保在执行此操作之前，您已经完成了对其他分支的任何必要操作和备份。

bash
git remote prune origin
这个命令会删除您本地记录的远程分支，在下次拉取时它们将不再被获取。

通过这些步骤，您将只拉取远程主分支的更新，并且只保留远程主分支的本地引用。如果您有其他特定需求或情况，请随时告诉我，我将竭诚为您服务。
