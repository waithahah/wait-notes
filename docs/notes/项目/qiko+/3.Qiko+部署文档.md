# Qiko+ 部署文档

## 1. 概述
- 项目名称：Qiko+
- 描述：ai应用快速构建平台
- 版本：1.1.0
- 作者：qkos

## 2. 前提条件
### 硬件要求
- CPU：8核
- 内存：24GB
- 存储：500GB SSD
- 网络带宽：2Gbps

> 以上配置为最低要求，实际部署时根据部署环境的实际情况调整。
### 软件要求
- 操作系统：Linux 操作系统
- 必要软件：Docker 19.03 或更高版本 Docker Compose 1.25.1 或更高版本

## 3. 准备工作
### 环境设置
- 安装Docker：`sudo apt install docker`
- 安装Docker Compose：`sudo apt install docker-compose`
> 上述命令适用于基于Ubuntu的Linux系统。在其他Linux发行版中，安装Docker和Docker Compose的步骤可能会有所不同，具体取决于该发行版使用的包管理工具和软件源。

## 4. 推荐配置
### 推荐操作系统
- CentOS Linux 7 x86_64
### 推荐 docker 版本
- Docker version Docker 19.03 或更高版本 Docker Compose 1.25.1 或更高版本
### 推荐浏览器
- Google Chrome 96+

## 5. 部署
1. 先查看 qiko+部署信息 文件夹下的部署文件 Qiko+ 部署文档.md 根据部署文件部署
2. 获取docker镜像并加载到系统中，镜像包在：qiko+部署信息\docker镜像包\_x86_64 文件夹内
> 先把镜像包传到服务器某个目录下，执行命令加载镜像 `docker load -i xxx(具体镜像包名字).tar.gz`
3. 修改 qiko+部署信息\docker-qiko 目录下的 .env.example 文件为 .env 文件
4. [可选]如果使用 qiko+ 提供的pg数据库，无需提前建数据库 qiko,否则需要提前在pg数据库建立数据库 qiko
> 如有需要提前在pg数据库建立数据库 qiko：创建脚本可使用 qiko+部署信息\docker-qiko\startupscripts\create_user.sql
5. [可选]根据实际环境信息修改 .env 文件 里面的 环境变量（如无需修改，则使用默认配置，可跳过这一步）
6. 在 qiko+部署信息\docker-qiko 目录下 执行 `docker compose up -d` 即可部署成功

## 6. 数据初始化
- 进入 api 服务容器内： docker exec -it docker-legacy-api-1 bash
- 执行以下命令，当控制台出现 "初始化工具的标签成功!" 即可
```bash
    poetry run python -m flask init-tool-tag-lables
```
