# 专业名词
* `上下文大小`: 大模型名字中带的“8k”或“32k”通常指的是模型的上下文窗口大小，也称为上下文长度或序列长度。具体来说，这是模型在处理输入文本时可以同时考虑的最大标记（token）数量。标记可以是单词、子词或字符，具体取决于模型的分词方式。
* `上下文窗口大小`: 上下文窗口大小是指模型一次性能够处理和记住的最大标记数量。标记（token）是模型处理的最小单位，可以是单词、子词或字符。上下文窗口大小（如8k或32k）指的是模型在单次输入处理中能够考虑的最大token数量，而不是整个聊天会话的总token数。在对话过程中，模型会优先保留最近的对话内容，确保在生成回复时能够基于最新的上下文进行。
* `token`: Token是模型处理文本的基本单位，可以是单词、子词或字符。上下文窗口大小（如8k或32k）指的是模型在一次处理过程中能够考虑的最大token数量。分词方式和token的数量直接影响模型的计算需求和性能，因此在设计和使用大型语言模型时，理解token的概念和作用是非常重要的。具体取决于模型使用的分词方法。

# 分类
* `llm`: 是一种基于对数双线性模型的语言模型。它使用一个双线性函数来建模词与上下文之间的相互作用，通过最大化给定上下文的条件概率来预测下一个词。LLM的优点在于可以捕捉到较远距离的上下文信息，从而获得更好的语言模型性能。
* `Text Embedding`: 是一种将文本映射到低维向量空间的方法。它把每个词或文档表示为一个稠密的向量，使得语义相似的词或文档在向量空间中距离较近。Text Embedding模型的代表性方法是Word2Vec和GloVe等。这些模型常用于计算词语的相似度、推荐系统、情感分析等任务。
* `Rerank Model`: 是一种用于搜索或推荐系统中的模型，用于对候选项进行重排序（reranking）。当搜索引擎或推荐系统返回一个初始的候选列表时，Rerank Model会对这些候选项进行重新排序，以提供更精确或个性化的结果。

Rerank Model通常是在初始列表上应用机器学习算法或深度学习模型，利用各种特征和信号来评估每个候选项的相关性、质量或用户兴趣。这些特征可以包括文本相似度、关键词匹配、用户历史行为、位置信息等等，具体根据不同的应用场景而定。

Rerank Model的目标是通过重新排序来提高搜索结果或推荐的准确性和用户满意度。它可以帮助过滤掉不相关或低质量的候选项，并将最相关或最合适的候选项呈现给用户，从而提供更好的搜索体验或推荐推断。

总之，Rerank Model是一种对搜索或推荐结果进行重排序的模型，通过机器学习算法或深度学习模型来评估和调整候选项的顺序，以提供更准确和个性化的结果。
* `Speech-to-text Model`: 语音转文本能力
* `Text-to-speech Model`: 文本转语音能力
