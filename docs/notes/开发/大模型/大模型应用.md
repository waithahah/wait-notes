# 模型调用
在实现LLM（Large Language Model）调用的核心方法时，支持流式和同步返回有以下区别：

1. 流式调用：
    * 流式调用允许在模型生成结果时持续输出部分结果。这对于处理大量数据或需要逐步处理结果的场景非常有用。
    * 在流式调用中，可以将输入数据分块发送到模型，然后逐块获取输出结果，这样可以降低整体延迟，并且节省内存和带宽资源。
    * 流式调用通常需要客户端和服务端之间建立长连接，以便在处理过程中进行双向通信。
2. 同步返回：
    * 同步调用是指客户端发送请求后等待服务器处理完成并返回最终结果后才继续执行后续操作。
    * 在同步调用中，客户端需要等待服务端完成整个计算过程，然后一次性获取完整的结果。
    * 同步调用适用于对实时性要求不高，且一次性获取全部结果即可满足需求的场景。

在实际应用中，选择流式调用还是同步返回取决于具体需求。如果需要实时获取部分结果或者处理大量数据且希望快速得到部分处理结果，那么流式调用是一个更好的选择。而如果只关心最终结果且对实时性要求不高，同步返回可能更加简单直接。
