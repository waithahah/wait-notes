# 遇到的问题
1. 遇到提交前校验的问题，可以先点githubdestop的冲突弹窗右上角的x
2. 然后手动修改完代码，提交代码
3. 构建镜像失败，可以不使用缓存先构建一次
```bash
docker build --no-cache -t qiko-plus-test:test .
```

4. 后端需要重新在本地lock一下依赖，在提交到线上
```bash
poetry lock
```
