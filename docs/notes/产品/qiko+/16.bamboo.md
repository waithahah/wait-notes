```sh
WEB_IMAGE=qkos/qiko-plus/web
API_IMAGE=qkos/qiko-plus/api
VERSION="${bamboo.version}"


current_time=$(date +"%Y%m%d%H%M%S")
echo "Current time in the format: $current_time"
workingdir=${bamboo.build.working.directory}
echo "dizhi : $workingdir"

docker tag $WEB_IMAGE:$VERSION $WEB_IMAGE:tmp
docker tag $API_IMAGE:$VERSION $API_IMAGE:tmp

docker build -t $WEB_IMAGE:$VERSION ./web
docker build -t $API_IMAGE:$VERSION ./api

docker image rm -f $WEB_IMAGE:tmp
docker image rm -f $API_IMAGE:tmp
```

```sh
WEB_IMAGE=qkos/qiko-plus/web
API_IMAGE=qkos/qiko-plus/api
VERSION="${bamboo.version}"

if [ "$VERSION" = "release" ]; then
    docker save $WEB_IMAGE:$VERSION $API_IMAGE:$VERSION > qikoplus-tmp.tar
    gzip qikoplus-tmp.tar
    echo "Packaging completed for prod version"
else
    echo "Not packaging for non-prod version"
fi
```

# 版本管理：
## 分支管理：
1. `main`分支：最近同步的dify最新版
2. `master`分支：qiko+最新稳定版，后续部署包将从这个分支产生
3. `dev`分支：研发分支
## tag管理
1. 每次版本发布后将生成对应的tag，管理所有已发布版本

# 互联网研发环境和正式环境更新步骤：
## 互联网研发环境[172.21.43.254](http://172.21.43.254/)
1. `dev`分支提交代码即可自动更新
## 互联网正式环境
1. bamboo上手动点击 `master`分支 的构建计划
2. 构建完成后，手动点击 `master`分支 的部署计划
3. 即可自动更新正式环境
