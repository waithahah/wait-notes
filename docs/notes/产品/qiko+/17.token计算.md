# 实现方式：

## 前置条件： 

    * 模型类型 `llm` `text-embedding` 可用

    * 大模型有提供 token 计算的接口（如通义千问）

    * qiko+有配置大模型上下文大小 `context_token`（必需）和大模型单次返回内容 `max_token` 的token大小（非必需）

## 上下文token计算公式：

    rest_tokens(剩余token) = model_context_tokens(配置的模型上下文大小) - max_tokens（大模型单次返回token大小） - prompt_tokens（提问内容token大小）

## 截取策略

### 截取条件
    1. 有系统提示词 且 无记忆
        最近一次聊天（系统提示词 + 用户提问内容）的 token < 0 才需要截取
    2. 有系统提示词 且 无记忆
        最近一次聊天（系统提示词 + 用户提问内容）的 token < 0 才需要截取
    3. 无系统提示词 且 有记忆
        上下文的 token < 0 才需要截取
    4. 无系统提示词 且 有记忆
        上下文的 token < 0 才需要截取

### 截取策略
    1. 按时间线和上下文大小从前往后移除prompt_message
    2. 如果有系统提示词就保留系统提示词和用户提问内容，然后按上下文大小从前往后移除prompt_message
    3. 如果只剩下【系统提示词和用户提问内容】或者是只剩下【用户提问内容】，则对最后一个用户提示词的内容进行百分比裁剪，知道符合token限制大小

### 最后一个用户提示词的内容按百分比移除方式

    1. 移除公式：percentage_to_remove = (prompt_tokens + max_tokens - model_context_tokens) / prompt_tokens * 100

    2. 确保百分比合理:如果percentage_to_remove小于0，则不需要移除任何prompt_tokens。如果percentage_to_remove大于等于100%，则需要移除所有prompt_tokens。

    3. 截取prompt_tokens：prompt_tokens -= (percentage_to_remove / 100) * prompt_tokens

    4. 确保rest_tokens >= 0: 再次计算rest_tokens并检查是否大于等于0: rest_tokens = model_context_tokens - max_tokens - prompt_tokens

> 说明：不满足前置条件的模型，不做截取
